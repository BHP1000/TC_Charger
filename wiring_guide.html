CAN Bus — WCMCU-230 (SN65HVD230)

  1. ESP32-S3 GPIO 4 → WCMCU-230 CTX
  2. ESP32-S3 GPIO 5 → WCMCU-230 CRX
  3. ESP32-S3 3V3 → WCMCU-230 3V3
  4. ESP32-S3 GND → WCMCU-230 GND
  5. WCMCU-230 CANH → Charger CAN High
  6. WCMCU-230 CANL → Charger CAN Low
  7. 120 Ω resistor across WCMCU-230 CANH and CANL
The charger has an internal 120 Ω terminator. Add one resistor at the WCMCU-230 end only. Firmware bitrate: 250 kbps.

OLED — SH1106 1.3" 128×64

GPIO 8 and GPIO 9 are the I2C bus. The OLED and the env sensor below share these two wires.
  1. ESP32-S3 GPIO 8 → OLED SDA
  2. ESP32-S3 GPIO 9 → OLED SCK (pin is labelled SCK on the module; it carries the I2C clock)
  3. ESP32-S3 3V3 → OLED VCC
  4. ESP32-S3 GND → OLED GND
If the screen stays blank, the I2C address may be 0x3D instead of 0x3C. Check the SA0 solder pad on the back of the module.

Env Sensor (outside box) — AHT20 + BMP280

Mounts outside the battery box on the Main Brain board. Shares the same I2C bus as the OLED — just run SDA and SCL to both modules in parallel. This sensor reads ambient air around the charger electronics.
AHT20 I2C address: 0x38  |  BMP280 I2C address: 0x76 or 0x77 — no conflict with OLED (0x3C).
  1. ESP32-S3 GPIO 8 → AHT20+BMP280 SDA (same wire as OLED SDA)
  2. ESP32-S3 GPIO 9 → AHT20+BMP280 SCL (same wire as OLED SCL)
  3. ESP32-S3 3V3 → AHT20+BMP280 VCC
  4. ESP32-S3 GND → AHT20+BMP280 GND

RS485 — 3.3 V Auto-Direction Module

Use the 3.3 V auto-direction module (pins: VCC TXD RXD GND on one side, A B GND on the other). It handles TX/RX switching automatically — no DE or RE wire needed from the ESP32-S3. Bus baud rate: 115200 8N1.
  1. ESP32-S3 GPIO 17 → RS485 module TXD
  2. ESP32-S3 GPIO 18 → RS485 module RXD
  3. ESP32-S3 3V3 → RS485 module VCC
  4. ESP32-S3 GND → RS485 module GND
  5. RS485 module A → RS485 Bus Wire A+
  6. RS485 module B → RS485 Bus Wire B−

OLED Page Buttons

Two momentary push buttons wired active-LOW. Firmware uses INPUT_PULLUP — no external resistors needed. Press NEXT to advance one page; press HOLD to toggle display freeze (header shows [HLD] when frozen).
  1. ESP32-S3 GPIO 6 → NEXT button pin 1
  2. ESP32-S3 GND → NEXT button pin 2
  3. ESP32-S3 GPIO 7 → HOLD button pin 1
  4. ESP32-S3 GND → HOLD button pin 2
Debounce: 50 ms in firmware. Any standard 6 mm or 12 mm momentary tactile switch works. No orientation — either pin to GPIO, other pin to GND.

Future expansion (resistor ladder): If more buttons are needed later without using more GPIO pins, multiple buttons can share a single analog pin through a resistor voltage divider ladder. Each button connects a different resistor value to ground, producing a unique ADC voltage. One GPIO (ADC-capable) can then identify 4–8 distinct button presses. GPIO 6 and GPIO 7 can be freed for this if needed — the only firmware change required is replacing the digital read with an ADC threshold check.
GPIO 48 drives the onboard WS2812B RGB LED. No external LED wiring needed.
Board: ESP32-C3 Super Mini. All pins are 3.3 V.
GPIO 9 is also the boot button. It is used here for I2C SCL. The I2C pull-up resistors (inside the sensor module) keep it HIGH at power-on so boot mode is not triggered. Safe for normal use.

Env Sensor (inside battery box) — AHT20 + BMP280

Mounts inside the battery box to measure cell-level heat and humidity. Compare these readings against the outside sensor on the Main Brain to see the differential.
  1. ESP32-C3 GPIO 8 → AHT20+BMP280 SDA
  2. ESP32-C3 GPIO 9 → AHT20+BMP280 SCL
  3. ESP32-C3 3V3 → AHT20+BMP280 VCC
  4. ESP32-C3 GND → AHT20+BMP280 GND

RS485 — Manual DE/RE Module (MAX485 or equivalent)

The Temp Node must use a manual DE/RE RS485 module — NOT the auto-direction module used on the Main Brain. This is required because the C3 acts as Modbus RTU master to the JK-BMS and must precisely control when the line switches between transmit and receive. GPIO 6 is the DE/RE control pin (HIGH = transmit, LOW = receive). Tie the DE and RE pins on the module together and connect both to GPIO 6.

Common modules: MAX485, MAX3485, SP3485, SN75176 breakout boards — any that expose separate DE and RE pins. Bus baud rate: 115200 8N1.
  1. ESP32-C3 GPIO 5 → RS485 module DI (TXD / Data In)
  2. ESP32-C3 GPIO 10 → RS485 module RO (RXD / Receiver Out)
  3. ESP32-C3 GPIO 6 → RS485 module DE and also RS485 module RE (tie DE and RE together)
  4. ESP32-C3 5V → RS485 module VCC (MAX485 is a 5 V device — do NOT connect to 3V3)
  5. ESP32-C3 GND → RS485 module GND
  6. RS485 module A → RS485 Bus Wire A+
  7. RS485 module B → RS485 Bus Wire B−

  2. NTC 2 mid-point → ESP32-C3 GPIO 1

BMS Model: JK-B2A20S20P — 20S Li-Ion, 73 Ah design capacity.
Protocol: JK BMS RS485 Modbus V1.0, selected in the JK app under UART Protocol. Device address 001 (0x01). Baud rate 115200 8N1.

CAN / RS485 Port — Connector Identification

The BMS has two ports on the same side. Use the one labelled CAN / RS485 — NOT the GPS port (labelled GND RX TX VCC).

The CAN/RS485 connector has 4 physical pin positions but only 3 are used. The labels are printed on the PCB silkscreen next to the connector: GND · A/H · B/L. The 4th pin position has no label and is NC (not connected) — leave it empty.

A/H = RS485 A (non-inverting, +).   B/L = RS485 B (inverting, −).

BMS to RS485 Bus

  1. BMS CAN/RS485 pin GND → RS485 Bus GND (common ground with all boards)
  2. BMS CAN/RS485 pin A/H → RS485 Bus Wire A+
  3. BMS CAN/RS485 pin B/L → RS485 Bus Wire B−
  4. BMS CAN/RS485 pin 4 (unlabelled) — leave unconnected (NC)
The BMS connects directly to the shared A+/B− RS485 bus — no separate RS485 module is needed on the BMS side. The JK-B2A20S20P has a built-in RS485 transceiver on that port.

Testing BMS Communication (Standalone — No Main Brain Required)

You do NOT need the Main Brain connected to test BMS communication. The Temp Node (C3) handles all Modbus traffic to the BMS independently. Procedure:

1. Power the C3 via USB to your PC only (Main Brain left disconnected).
2. Wire C3 RS485 module A+ / B− / GND to BMS CAN/RS485 A/H · B/L · GND.
3. Open a Serial Monitor at 115200 baud on the C3's COM port.
4. Every 5 seconds you will see one of two outputs:
   • Success: [JK-BMS] SOC=xx% Pack=xx.xxxV I=x.xxxA SOH=xx% Cyc=x Alm=0x00000000
   • Failure: [JK-BMS] cell voltage read FAILED

If you get FAILED, swap A and B wires — reversed polarity is the most common cause and causes no damage.

Pin Identification With a Multimeter (If Labels Are Hard to Read)

  1. Set multimeter to continuity or resistance mode.
  2. Probe between each connector pin and the battery pack negative (B−) terminal on the BMS. The pin that shows continuity (near 0 Ω) is GND.
  3. The remaining two labelled pins are A/H and B/L. Connect them to bus A+ and B− respectively. If comms fail, swap them — no damage will result from reversed RS485 polarity.

Pin Identification When BMS Is Inside a Housing (No Silkscreen Visible)

If the BMS is already mounted inside a battery pack housing and you cannot see the PCB silkscreen, use this procedure to identify pins entirely from the outside.
  1. Locate the correct connector. The CAN/RS485 connector is the 4-pin JST-style socket. The GPS connector is a different 4-pin socket. If both look identical from outside the housing, check which cable exits toward the battery wiring harness side vs. the communication side — or count positions: the CAN/RS485 socket is the one with 3 wired pins and 1 empty position.
  2. Find GND without opening the housing. Set your multimeter to DC voltage. Touch the negative probe to the pack's negative output terminal (the large B− power lug) and touch the positive probe to each of the 4 connector pins in turn. The pin that reads 0 V (or within a few millivolts) is GND. All other pins will float or show a small RS485 bias voltage.
  3. Confirm GND with continuity if you have access to pack negative. Switch to continuity mode and probe between each pin and the B− lug. GND will beep; the others will not.
  4. Pin order reference (JK-B2A20S20P, looking into the connector face from outside the housing): The standard factory pinout is [NC] [B/L] [A/H] [GND] from left to right — but do not rely on this without verifying with the multimeter steps above, as connector orientation can vary by housing orientation.
  5. Identify A and B. Once GND is confirmed, connect the remaining two pins to the RS485 bus A+ and B− in any order. Power the Temp Node and watch the serial monitor. If you see [JK-BMS] cell voltage read FAILED, swap the two wires. No damage occurs from reversed RS485 polarity.
  6. Mark the wires with tape labels (GND / A+ / B−) before reassembling the housing so you do not need to repeat this procedure.
The 4th pin position is always NC — it carries no signal regardless of housing orientation. If you have found GND and still have 2 remaining active pins plus 1 that reads floating with no resistance path to anything, that floating one is NC.
Three devices share one RS485 bus: Main Brain (master), Temp Node (slave — also Modbus master to BMS), and JK-BMS (Modbus slave). Run a single twisted pair A+/B− between all three plus a common GND wire.

RS485 Bus Topology

All three devices connect to the same two wires (A+ and B−) in a daisy-chain or star arrangement. A+ connects to A+ on all devices; B− connects to B− on all devices. GND is also shared across all three.

The custom binary protocol packets from the Main Brain start with 0xAA — the JK-BMS ignores these because its Modbus parser only responds to packets addressed to 0x01 using Modbus framing.
The Modbus packets from the Temp Node start with 0x01 (BMS address) — the Main Brain ignores these because its custom packet parser only looks for the 0xAA sync byte.
All three devices coexist on the same wires without any conflicts.

Wiring — All Three Nodes

  1. Main Brain RS485 module A → Bus A+
  2. Main Brain RS485 module B → Bus B−
  3. Temp Node RS485 module A → Bus A+
  4. Temp Node RS485 module B → Bus B−
  5. JK-BMS CAN/RS485 A/H → Bus A+
  6. JK-BMS CAN/RS485 B/L → Bus B−
  7. Main Brain GND → Temp Node GND → JK-BMS CAN/RS485 GND (one shared GND wire along the whole bus run)
RS485 is wired straight-through — A to A, B to B across all nodes. It is not crossed. In the EIA-485 standard A is the non-inverting (+) line and B is the inverting (−) line on every device. Crossing A and B inverts the signal and the bus will not communicate.

CAN — Main Brain to Charger

  1. Main Brain WCMCU-230 CANH → Charger CAN High
  2. Main Brain WCMCU-230 CANL → Charger CAN Low
Firmware CAN bitrate: 250 kbps. The RS485 bus runs at 115200 8N1.
Sensor: 100K NTC 3950 Thermistor — Beta value 3950, nominal resistance 100 kΩ at 25°C.
Each sensor uses a measured ~95 kΩ fixed series resistor to form a voltage divider. The firmware constant NTC_SERIES_OHMS is set to 95000 to match the actual measured value of the 1W resistors in use.

Each NTC Thermistor (repeat × 5)

  1. ESP32-C3 3V3 → NTC leg A
  2. NTC leg B → GPIO pin (0, 1, 2, 3, or 4) and also to one leg of the 95 kΩ series resistor
  3. Other leg of series resistor → GND
The ADC reads the mid-point voltage between the thermistor and the fixed resistor. At 25°C the thermistor is 100 kΩ and the resistor is ~95 kΩ, so the mid-point sits slightly above 1.65 V. The Steinhart-Hart / Beta equation in firmware compensates for this automatically using the exact 95000 Ω constant.

Which GPIO for each sensor

  1. Sensor 1 → GPIO 0
  2. Sensor 2 → GPIO 1
  3. Sensor 3 → GPIO 2
  4. Sensor 4 → GPIO 3
  5. Sensor 5 → GPIO 4

GPS Module — HGLRC M100 / M100 PRO

The GPS module connects directly to the Main Brain (ESP32-S3). It uses UART2 for NMEA sentences and the onboard QMC5883L compass uses the shared I2C bus. Do not connect the GPS module to the BMS GPS port — that port is a passthrough for a different system and is not used here.

UART Connection (NMEA / GPS data)

Firmware uses Serial2 on GPIO 15 (RX) / GPIO 16 (TX). Module baud rate: 115200. The M100 connector is a 6-pin SH1.0 (1 mm pitch). The M100 PRO uses the same pinout. You will need an SH1.0 6-pin to bare-wire adapter or a compatible pigtail cable.
  1. ESP32-S3 3V3 → GPS module VCC (SH1.0 pin 1 — red wire)
  2. ESP32-S3 GND → GPS module GND (SH1.0 pin 2 — black wire)
  3. ESP32-S3 GPIO 15 (RX2) → GPS module TX (SH1.0 pin 3)
  4. ESP32-S3 GPIO 16 (TX2) → GPS module RX (SH1.0 pin 4)
Pins 5 and 6 on the SH1.0 connector are the I2C lines for the QMC5883L compass (SDA / SCL). See section below — these connect to the shared I2C bus, NOT to separate pins.

Compass — QMC5883L (built into M100 / M100 PRO)

The QMC5883L is soldered onto the GPS module and communicates over I2C (address 0x0D). It shares the same I2C bus as the OLED and env sensor. No additional GPIO pins are needed — just connect pins 5 and 6 of the SH1.0 connector to the existing I2C bus wires.
  1. GPS module SDA (SH1.0 pin 5) → ESP32-S3 GPIO 8 (shared I2C SDA)
  2. GPS module SCL (SH1.0 pin 6) → ESP32-S3 GPIO 9 (shared I2C SCL)
I2C addresses in use: OLED = 0x3C, AHT20 = 0x38, BMP280 = 0x76/0x77, QMC5883L = 0x0D. No conflicts.

M100 vs M100 PRO — Compatibility Note

Both modules use the same 6-pin SH1.0 connector with the same pin order and the same firmware settings. The PRO variant has a larger ceramic patch antenna and better sensitivity in poor sky conditions. Firmware detects the GPS automatically via NMEA sentences — no code change needed to switch between them.

JK-BMS GPS Port — Do Not Use

The JK-BMS has a 4-pin port labelled "GPS" (GND · RX · TX · VCC). This is not where the GPS module connects in this system. The BMS GPS port is for a different accessory. Leave it unplugged.

Range Estimation

Firmware calculates remaining range as: remaining_mAh × pack_V ÷ 1000 ÷ consumption_Wh_per_km. Default consumption constant is 20 Wh/km (defined as GPS_CONSUMPTION_WH_PER_KM in gps_driver.h). Adjust this value to match your scooter's real-world consumption once you have ride data. The result is shown on the GPS OLED page and broadcast over BLE.
