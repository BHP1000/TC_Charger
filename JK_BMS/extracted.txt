JK BMS RS485 Modbus General Protocol (V1.0)
Translated from Chinese Original — Technical/Literal Translation for Protocol Implementation
The JK BMS RS485 Modbus General Protocol uses a master-slave request-response communication model. Only the
master can initiate requests via the unique slave address, and the BMS (slave) responds according to the master's
requests, implementing half-duplex communication. This protocol only allows the master to initiate requests while the
slave responds passively; therefore, the slave will never actively occupy the communication line, preventing data
collisions.

3) Data Area
The data area varies depending on the function code and data direction. The data may be combinations such as "starting
register address + number of registers to read", "register address + operation data", or "starting register address +
number of registers to operate + data length + data", etc. Detailed data area formats for different function codes are
explained in the "Function Code Analysis" section.
4) CRC Checksum
The CRC checksum is used to ensure the correctness and integrity of data transmission.
III. Error Feedback
Address and CRC checksum errors will not receive any data feedback from the slave. Other errors will return error codes
to the master. The second byte of the data frame is the original function code with 0x80 added, indicating a request error
(illegal function code, illegal data value, etc.). The error data frame format is:

IV. Information Transmission Process
When a communication command is sent by the master to the slave, the slave whose address matches the address code
sent by the master receives the communication command. If the CRC checksum is correct, the slave executes the
corresponding operation and returns the execution result (data) to the master. The return message contains the address
code, function code, post-execution data, and CRC checksum. If the address does not match or the CRC checksum is
incorrect, no information is returned.

⚠ Implementation Notes for Charger Integration:
All register offsets within a base address group are byte offsets, not Modbus register numbers. Divide byte offset
by 2 to get the Modbus register address for 16-bit registers.
Multi-byte values are transmitted in Big-Endian (high byte first) format as per standard Modbus convention.
CRC checksum uses CRC-16/Modbus (polynomial 0xA001), transmitted low byte first.
Temperature values in 0.1°C resolution: divide raw value by 10 to get degrees Celsius. Signed types (INT16/INT32)
support negative temperatures.
Current values (INT32 at 0x0098) are signed: positive = charge, negative = discharge.
Always check the Alarm Status Register (0x00A0) before and during charging operations for safety-critical fault
detection.
